# Feature Carousel Block - Test Content

## How to Use This Block

Add the following content structure to your document to create the feature carousel:

### Example Content Structure:

```
| Feature Carousel |
| bolt | Lightning Fast Development | Generate production-ready EDS blocks in seconds, not hours. Our AI understands your requirements and delivers optimized code instantly. | Instant code generation • Real-time preview • One-click deployment | [image: /path/to/rocket-image.png] |
| bolt | Smart Code Quality | Automated testing and quality checks ensure your blocks meet all standards. Built-in accessibility compliance and performance optimization. | WCAG 2.1 AA compliant • Performance optimized • SEO friendly | [image: /path/to/quality-image.png] |
| bolt | Seamless Integration | Works perfectly with your existing EDS setup. No configuration needed - just drop in and start using immediately. | Zero configuration • Works everywhere • Full documentation | [image: /path/to/integration-image.png] |
```

### Column Structure:

1. **Column 1**: Badge icon name (e.g., "bolt") - optional, leave blank if not needed
2. **Column 2**: Slide title (required)
3. **Column 3**: Slide description (required)
4. **Column 4**: Feature bullets separated by `•` (optional)
5. **Column 5**: Image for visual panel (optional)

### Tips:

- Use 3-5 slides for best results
- Keep titles short and impactful (3-6 words)
- Descriptions should be 15-25 words
- Use 2-4 feature bullets per slide
- Images work best at 600x450px or similar aspect ratio

### Navigation:

- **Mouse**: Click arrow buttons or dots
- **Keyboard**: Use Left/Right arrow keys
- **Touch**: Swipe left/right on mobile devices

### Accessibility:

- All navigation is keyboard accessible
- Screen reader support included
- Focus indicators are visible
- WCAG 2.1 AA compliant
